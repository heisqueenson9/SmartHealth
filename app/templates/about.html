PROJECT STATEMENT & REPORT

Academic Context

This system represents my final year capstone research for my degree in Information Technology within the Department of Computer Science at the University of Ghana.

Project Purpose

The objective of my study was to explore how modern information technology can augment clinical decision making in Ghana. With medical facilities facing high patient volumes, I wanted to build a prototype that uses machine learning to identify disease patterns before they become critical.

My system analyzes clinical biomarker data — which includes things like hematocrit and troponin levels — to provide instant diagnostic labels. I believe that such tools could one day support practitioners in rural areas where specialist access is limited.

Technical Methodology

Phase 1
Data Engineering
Processing 2,837 clinical records through stratified splits and feature scaling for optimal model training.
Phase 2
Model Selection
Comparing Support Vector Machines and Random Forest architectures to maximize weighted F1 performance scores.
Phase 3
Inference Engine
Developing a Flask based web interface to serve model predictions in a real time clinical simulation environment.
Phase 4
Optimization
Fine tuning class weight balances to improve detection of minority conditions like heart disease.

Technical Challenges

  • 01

    Handling data imbalance was my primary hurdle — specifically for heart disease data points which were significantly fewer than diabetes records.

  • 02

    Establishing a robust cross validation strategy ensured that accuracy scores were not inflated by overfitting to the training samples.

  • 03

    Managing the integration between the machine learning backend and a modern web interface while ensuring low inference latency.
